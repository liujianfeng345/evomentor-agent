# Skill: action-empty-result-detect

## 触发条件
当系统执行 periodic_check 或 user_message 动作后，日志中未记录任何结果，或产出数据为空时触发。若连续多次（如3次以上）结果均为空，则优先触发本技能。

## 行为规则
## 1. 检测方法
- 检查动作执行后的日志输出，确认是否存在结果记录。
- 对比动作输入和输出，验证是否缺失关键字段。
- 检查日志收集流程是否完整，确认是否因日志级别过滤、采集中断等原因导致结果未被记录。
- 若连续多次（如3次以上）结果均为空，则判定触发。

## 2. 修复建议
- 审查动作实现代码，确保每个分支都有明确的返回值或日志输出。
- 在动作执行关键节点增加日志输出，记录中间结果或关键字段（如生成摘要）。
- 检查日志收集管道是否正常，避免数据丢失或截断。
- 对空结果情况添加默认处理或异常告警，如增加异常捕获并记录错误。

## 3. 相关案例
- 经验[50]：所有 periodic_check 和 user_message 的结果均为空，需检查动作实现或日志收集流程。
- 经验[40]：user_message 和 reflect 触发后未记录任何分析结果或行动，需优化事件处理流程。
- 案例1：periodic_check 返回空列表，经查是数据库查询条件错误，修复后恢复。
- 案例2：user_message 处理后无输出，原因是消息解析失败未记录错误，增加异常捕获后解决。
