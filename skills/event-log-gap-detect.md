# Skill: event-log-gap-detect

## 触发条件
当系统检测到 periodic_check、user_message 或 reflect 等关键动作的事件结果字段为空或缺失时触发

## 行为规则
1. 检测方法：扫描最近24小时内的所有事件日志，检查 periodic_check、user_message、reflect 等关键动作的结果字段是否为空或未记录。若连续3次以上结果为空，则判定为日志不完整或存在潜在问题。
2. 修复建议：检查对应动作的实现代码，确保每个事件处理函数在结束时输出非空结果或错误信息；同时检查日志收集流程，确认日志是否被正确写入存储，避免因异步或缓存导致记录丢失。
3. 相关案例：用户 [50] 和 [40] 中，periodic_check 和 user_message 的结果均为空，reflect 后也无分析输出，表明日志记录或动作产出存在系统性缺陷，需定位到具体动作模块。
