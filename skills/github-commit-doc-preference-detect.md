# Skill: github-commit-doc-preference-detect

## 触发条件
当用户最近的 GitHub 提交分析显示，提交内容中 Markdown 文档、文本文件、图片等非可执行代码文件占比超过 50% 时触发（基于最近 30 次提交或最近 30 天提交记录分析）

## 行为规则
1. 检测方法：分析用户最近 30 次或最近 30 天 GitHub 提交记录，统计 Markdown、文本文件、图片等非可执行代码文件与代码文件（如 .py, .js, .java 等）的比例。若文档类提交占比超过 50%，则标记为文档偏向。
2. 修复建议：
   - 若用户目标为提升编码能力，建议增加代码提交比例（如 Python、JavaScript、Java 等），将知识文档转化为可执行的代码示例或项目，并增加可执行代码的提交。
   - 若用户目标是知识管理或技术写作，则当前模式合理。
3. 相关案例：用户提交了大量 Markdown 学习笔记或技能文档，但缺少对应代码实现，持续 3 次以上类似模式。建议将笔记中的算法、API 调用等转化为可运行的代码片段并提交。
