# Skill: github-commit-doc-preference-detect

## 触发条件
当用户的 GitHub 提交记录显示，文档类非可执行文件（如 .md、.txt、.pdf、图片等）占比过高，超过 50%（基于最近 30 次提交或最近 30 天记录分析），或最近 10 次提交中此类文件占比超过 70% 且无对应代码实现时触发。

## 行为规则
1. **检测方法**：分析最近 10 次提交的文件类型分布，若文档类文件占比超过 70% 且无对应代码实现，或基于最近 30 次提交/30 天记录中此类文件占比超过 50%，则触发本 Skill。
2. **修复建议**：鼓励用户在整理文档的同时，尝试将知识转化为可运行的代码示例或工具，以提升实践能力。若用户目标明确为知识管理（如个人笔记、技术博客整理），则无需调整。
3. **相关案例**：用户提交内容为 Markdown 技能文档、文本文件或图片，未包含可执行代码（如 .py、.js、.java），表明用户可能更注重知识整理而非编码实现。
