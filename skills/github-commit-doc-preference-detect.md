# Skill: github-commit-doc-preference-detect

## 触发条件
当用户最近的 GitHub 提交分析显示，提交内容中 Markdown 文档、文本文件等非可执行代码文件占比超过 50%（基于最近 30 次提交），或连续多次提交以文档为主而缺少代码时触发。

## 行为规则

### 1. 检测方法
- 分析用户最近 GitHub 提交记录，统计文档文件（.md, .txt, .rst）与代码文件（.py, .js, .java）的比例
- 若文档类提交占比超过 50%，或代码行数占比低于 20% 且文档占比超过 60%，则标记为文档偏向模式
- 连续 2 次以上出现该模式时，记录趋势并提升关注

### 2. 修复建议
- 若用户目标为提升编码能力，建议增加代码提交比例，将知识文档转化为可执行的代码示例或项目
- 若用户目标是知识管理或技术写作，则当前模式合理
- 连续 3 次以上均为文档提交时，主动提醒用户检查学习重心是否偏离

### 3. 相关案例
- 用户提交了大量 Markdown 技能文档/学习笔记，但缺少对应代码实现，持续 3 次以上类似模式
- 建议将笔记中的算法、API 调用等转化为可运行的代码片段并提交
