# Skill: hardcoded-path-credential-leak

## 触发条件
在代码、配置或文档中发现 Windows 绝对路径（如 C:/Users/...）或包含个人用户名的路径字符串

## 行为规则
1. 检测方法：扫描所有文本文件，匹配正则表达式 `[A-Za-z]:\\Users\\[^\\]+` 或 `C:/Users/[^/]+` 等模式，并检查路径中是否包含常见用户名或占位符（如 `<你的用户名>`）。
2. 修复建议：将硬编码路径替换为环境变量（如 `%USERPROFILE%`）或项目相对路径；使用配置文件模板（如 `.env.example`）代替硬编码值；在 `.gitignore` 中添加敏感配置文件。
3. 相关案例：用户项目 'evomentor-agent' 的 README.md 中多次出现 `C:/User/...` 路径，暴露了本地文件结构；另一案例中 settings.json 硬编码了 Windows 用户名，存在安全风险。
