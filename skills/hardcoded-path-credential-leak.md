# Skill: hardcoded-path-credential-leak

## 触发条件
当检测到代码或文档中包含 Windows 格式绝对路径（如 C:/Users/...）或包含用户名、本地文件结构等敏感信息时触发

## 行为规则
## 行为规则
1. **检测方法**：
   - 扫描代码、配置文件、README 等文本中是否包含 `[A-Za-z]:/Users/` 或 `[A-Za-z]:\\Users\\` 模式的路径。
   - 检查路径中是否包含常见的用户名占位符（如 `<你的用户名>`、`yourname`、`username`）或真实用户名。
   - 在版本控制（如 GitHub）提交内容中，检查是否包含 `.gitignore` 未忽略的敏感信息文件（如 `settings.json` 中的硬编码路径）。

2. **修复建议**：
   - 将所有硬编码的绝对路径替换为环境变量（如 `%USERPROFILE%`、`$HOME`）或相对路径。
   - 在配置文件中使用模板变量（如 `{{USERNAME}}`），并添加 `.gitignore` 忽略实际配置文件，仅提交模板文件。
   - 对于已经泄露的敏感信息，建议立即修改相关凭据或路径，并清理版本历史（如使用 `git filter-branch` 或 `BFG Repo-Cleaner`）。

3. **相关案例**：
   - 经验 [92]：`C:/User/...` 路径硬编码导致用户名泄露。
   - 经验 [80]：`C:/Users/<你的用户名>` 硬编码，暴露用户结构。
   - 经验 [72]：`settings.json` 中硬编码用户名 `<你的用户名`，存在安全风险。
