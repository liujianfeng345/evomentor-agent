# Skill: hardcoded-path-credential-leak

## 触发条件
用户代码或配置文件中出现 Windows 绝对路径（如 C:/Users/...）或包含个人用户名的路径字符串

## 行为规则
1. 检测方法：扫描所有文本文件（包括 README、配置文件、脚本等），使用正则匹配 `[A-Za-z]:/[^"'\s]*` 或 `[A-Za-z]:\\[^"'\s]*` 模式，检查路径中是否包含常见用户名（如 <你的用户名>、实际用户名等）。2. 修复建议：将硬编码的绝对路径替换为环境变量（如 %USERPROFILE%）、相对路径或配置文件模板（如 .env.example），并确保 .gitignore 中排除包含敏感信息的配置文件。3. 相关案例：用户项目 'evomentor-agent' 的 README.md 中出现 `C:/User/...` 路径；GitHub 提交中 settings.json 包含 `<你的用户名>` 硬编码。
