# Skill: github-doc-vs-code-pattern-recognition

## 触发条件
在 GitHub 提交分析中，连续多次（≥2次）提交内容主要为 Markdown、文档、配置文件，而非可执行代码

## 行为规则
1. 检测方法：统计单次提交中代码文件（如 `.py`, `.js`, `.java`）与文档文件（如 `.md`, `.rst`, `.txt`）的比例；若代码行数占比低于 20% 且文档行数占比超过 60%，则标记为文档倾向模式。
2. 修复建议：无需强制干预，但需记录趋势；如果连续 3 次以上均为文档提交，建议提醒用户检查学习重心是否偏离编码实践，可推荐代码练习或项目任务。
3. 相关案例：用户近期提交均为 Markdown 技能文档，未包含可执行代码，表明用户更注重知识整理而非实际开发，需持续关注是否影响编程技能提升。
