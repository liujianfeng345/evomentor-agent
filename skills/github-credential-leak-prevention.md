# Skill: github-credential-leak-prevention

## 触发条件
在 GitHub 提交分析中发现包含硬编码的用户名、路径、密钥等敏感信息

## 行为规则
1. 检测方法：扫描提交文件内容，识别硬编码的 Windows 用户名（如 `<你的用户名>`）、绝对路径（如 `C:\Users\...`）、API 密钥、密码等模式；使用正则或关键字匹配，标记包含敏感信息的文件和行号。
2. 修复建议：将所有硬编码的敏感信息替换为环境变量（如 `%USERNAME%`）或配置文件模板；在 `.gitignore` 中添加配置文件；使用 `git-secrets` 或 `pre-commit` 钩子防止后续提交。
3. 相关案例：用户在 `settings.json` 中硬编码了 Windows 用户名，导致个人信息暴露在公开仓库中，需立即清理提交历史并更新凭证。
