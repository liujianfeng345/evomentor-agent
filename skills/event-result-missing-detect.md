# Skill: event-result-missing-detect

## 触发条件
当 periodic_check 或 user_message 事件触发后，系统未记录任何有效结果或分析数据时

## 行为规则
1. 检测方法：检查所有 periodic_check 和 user_message 事件触发后的日志记录，确认是否有非空的结果输出（如分析结论、行动记录、状态变更等）。若连续多次结果为空，则判定为异常。
2. 修复建议：
   - 检查动作实现代码，确保每个事件处理函数在完成处理后返回或记录至少一个有效字段。
   - 检查日志收集流程，确认日志是否被正确写入、传输和存储，避免因日志丢失导致误判。
   - 在事件处理关键节点（如用户输入解析后、反思完成后）添加强制日志输出，即使无具体内容也应输出占位符（如 'no_action_taken'）。
3. 相关案例：
   - 案例1：user_message 触发后，日志仅记录 'received' 但无后续分析结果，实际是处理函数未执行解析逻辑。修复后增加对消息内容的分类和状态更新。
   - 案例2：periodic_check 多次触发但结果字段为空，经查是检查函数未正确连接数据源。修复数据源连接后，日志正常输出检查结果。
