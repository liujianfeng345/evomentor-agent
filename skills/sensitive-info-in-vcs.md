# Skill: sensitive-info-in-vcs

## 触发条件
在版本控制系统（如 GitHub）的提交历史、代码或文档中检测到敏感信息（如密码、API 密钥、个人路径、用户名等）

## 行为规则
1. 检测方法：分析提交内容（包括 diff 和文件内容），使用正则或关键词匹配常见敏感模式：硬编码的文件路径（如 C:/Users/...）、密码/令牌占位符、用户名、邮箱地址等。同时检查是否有配置文件（如 settings.json）包含未脱敏的个人信息。
2. 修复建议：立即从版本历史中移除敏感文件（使用 `git filter-branch` 或 BFG Repo-Cleaner），并更新 .gitignore 以排除本地配置文件。建议使用环境变量或配置模板来管理敏感信息，并启用 Git 提交前钩子（pre-commit hook）进行自动扫描。
3. 相关案例：用户提交的 settings.json 中包含硬编码的 Windows 用户名 `<你的用户名>`；README.md 中多次出现包含用户名的绝对路径，构成敏感信息泄露风险。
