# Skill: sensitive-info-in-vcs

## 触发条件
检测到版本控制仓库（如 GitHub）的提交或代码中包含敏感信息（如个人路径、用户名、密码、密钥、API token 等）。

## 行为规则
## 检测方法
1. 审查提交历史中的文件变更，检查新增或修改的内容是否包含常见的敏感信息模式（如 `C:/Users/`、`password=`, `api_key=`, `token=` 等）。
2. 特别关注配置文件（.json, .yaml, .env, settings.py 等）和文档文件（.md, .txt）。
3. 使用正则表达式匹配可能的泄露模式，并输出文件路径和具体行号。

## 修复建议
1. 立即从仓库历史中移除包含敏感信息的提交（使用 `git filter-branch` 或 BFG Repo-Cleaner）。
2. 将敏感信息替换为占位符或环境变量引用，并更新 `.gitignore` 避免未来提交。
3. 建议用户使用环境变量或密钥管理服务（如 GitHub Secrets）来管理敏感配置。
