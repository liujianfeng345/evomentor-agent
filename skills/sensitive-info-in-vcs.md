# Skill: sensitive-info-in-vcs

## 触发条件
当检测到代码仓库的提交历史或当前文件中包含可能泄露的敏感信息（如用户名、密码、API密钥、私钥、内部路径等）。

## 行为规则
### 1. 检测方法
- 遍历仓库所有提交历史，使用正则或关键词匹配敏感模式：`password\s*[:=]\s*['"][^'"]+['"]`、`api_key\s*[:=]\s*['"][^'"]+['"]`、`private_key`、`-----BEGIN RSA PRIVATE KEY-----` 等。
- 检查 `.gitignore` 是否忽略了常见的敏感文件（如 `.env`、`*.key`、`config.private.*`）。
- 分析提交信息中是否包含敏感词汇（如 `password`、`secret`、`token`）。

### 2. 修复建议
- **立即移除**：从当前分支中删除包含敏感信息的文件，并更新 `.gitignore`。
- **历史清理**：使用 `git filter-branch` 或 `BFG Repo-Cleaner` 从所有历史中彻底移除敏感信息。
- **轮换凭证**：如果泄露的是真实凭证（如 API 密钥），立即在服务端重新生成并更新。
- **审计通知**：通知项目成员，并审查是否有未授权的访问记录。

### 3. 相关案例
- GitHub 提交中 settings.json 硬编码了 Windows 用户名（`<你的用户名`）。
- 用户项目 README.md 中暴露了 `C:/User/...` 路径，包含用户名。
- 常见错误：将 `.env` 文件提交到公开仓库，导致数据库密码泄露。
